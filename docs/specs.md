# 高考志愿填报专业决策系统（MVP）
**Product Requirements Document（PRD）**

---

## 一、项目概述

### 1.1 项目背景
中国大陆高考志愿填报高度依赖历史录取数据、位次判断与专家经验。现有产品多停留在“查分/查线”层面，缺乏趋势判断与可解释性，难以支撑专业咨询场景。

本项目旨在构建一套**面向志愿填报机构的专业决策支持系统**，通过规则化算法与可视化解释，辅助顾问高效、可信地完成志愿方案设计。

### 1.2 产品定位
> 在你这个分数段里，哪些学校和专业更值得选，以及为什么。

- 专业工具，而非大众查分网站
- 规则驱动、可解释、可复用
- 面向咨询决策场景

### 1.3 目标用户
- 主要用户：志愿填报顾问、机构内部人员
- 次要用户（未来）：家长、考生（只读）

---

## 二、核心需求

### 2.1 核心目标
1. 快速生成冲 / 稳 / 保的院校-专业组合
2. 发现同分段内性价比高、趋势友好的选择
3. 支持咨询场景下的解释与对话

### 2.2 关键前提
- 平行志愿省份为主
- 以位次为第一决策依据
- 使用近三年数据
- 明确考虑趋势变化

---

## 三、核心功能

### 3.1 方案生成
**必填输入：**
- 省份
- 科类
- 高考分数
- 位次
- 批次
- 是否考虑民办 / 中外合作 / 独立学院

**可选输入：**
- 兴趣方向（最多 5 个标签）
- 职业倾向（就业 / 深造）
- 地域偏好

### 3.2 推荐输出
- 冲 / 稳 / 保三档推荐
- 每项包含：
  - 趋势修正后的参考位次
  - 相对价值评分（方案内有效）
  - 趋势提示
  - 捡漏弱提示

### 3.3 捡漏提示
- 基于趋势反差
- 弱提示，不做承诺
- 用于专家提醒

## 四、核心组件

### 4.1 数据组件
- User
- School
- Major
- AdmissionHistory
- Plan
- PlanItem

### 4.2 算法组件
1. 位次窗口筛选
2. 三年趋势判断
3. 趋势修正
4. 冲 / 稳 / 保分类
5. 相对价值评分
6. 捡漏提示

### 4.3 可视化组件
- 专业价值雷达图
- 位次分布云图
- 院校-专业关系图
- 专业画像卡
